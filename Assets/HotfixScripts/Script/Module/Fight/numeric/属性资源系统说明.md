# 战斗属性与资源系统说明

## 一、系统目的

当前战斗数值层分成两部分：

- `Property`：用于表示角色的最终属性，例如 `Attack`、`Defense`、`MaxHp`
- `Resource`：用于表示战斗中的当前资源，例如当前生命值、当前魔法值、当前体力值

这两层的职责不同，不应该混用。

## 二、核心结构

### 1. Property：属性

属性由 `CombatPropertySet` 管理，定义和计算逻辑在：

- `Number.cs`
- `StatDefinition.cs`

属性的典型特点：

- 有 `BaseValue`
- 有 `FinalValue`
- 可以被 `Modifier` 修正
- 适合承载“参与公式计算的值”

当前项目里已经存在的属性包括：

- `Attack`
- `Defense`
- `SpeedMultiplier`
- `RotationMultiplier`
- `MaxHp`

### 2. Resource：资源

资源由 `ResourceValue` 表示，定义在：

- `ResourceValue.cs`

资源的典型特点：

- 有当前值 `Value`
- 有最大值 `MaxValue`
- 有百分比 `Percent`
- 有变化事件，例如扣减、增加、归零

当前项目里已经存在的资源包括：

- `Hp`

### 3. Property 与 Resource 的关系

这套系统里，常见关系是：

- “最大值”属于 `Property`
- “当前值”属于 `Resource`

例如生命系统：

- `PropertyType.MaxHp` 表示最大生命值
- `ResourceType.Hp` 表示生命资源
- `Hp.Value` 表示当前生命值
- `Hp.MaxValue` 通常由 `MaxHp` 同步驱动

可以把它理解为：

- `MaxHp` = 血条长度
- `Hp.Value` = 血条当前剩余量

## 三、当前代码中的实际结构

### 1. 入口对象

战斗实体 `CombatEntity` 中持有：

- `CombatPropertySet properties`
- `ResourceValue hp`

也就是说，一个战斗单位会同时拥有：

- 一套属性容器
- 若干资源条

### 2. 当前血量与最大血量

在当前实现里：

- `hp` 是资源对象
- `MaxHp` 是属性
- `hp` 通过 `RegisterResource(ResourceType.Hp, PropertyType.MaxHp)` 绑定到 `MaxHp`

所以生命值不是一个单独的整数，而是拆成：

- 最大生命值：属性
- 当前生命值：资源

### 3. Modifier 的作用

属性支持临时修正，适合做：

- Buff
- Debuff
- 装备词条
- 技能临时加成

当前支持的修正类型：

- `Add`：直接加减固定值
- `AddPercent`：按百分比叠加
- `Multiply`：按倍率相乘
- `Override`：直接覆盖最终值

## 四、应该怎么使用

### 1. 什么适合放在 Property

以下内容适合定义为属性：

- 攻击力
- 防御力
- 最大生命值
- 最大魔法值
- 暴击率
- 暴击伤害
- 移动速度倍率
- 旋转速度倍率

判断标准是：

- 它会参与公式计算
- 它可能被 Buff、装备、被动、技能修正
- 它本身不需要“当前剩余量”

### 2. 什么适合放在 Resource

以下内容适合定义为资源：

- 当前生命值
- 当前魔法值
- 当前体力值
- 当前护盾值

判断标准是：

- 它有“当前值”和“最大值”
- 它会在战斗过程中被消耗、回复、清空

### 3. 一条常见规则

建议统一遵守下面这条规则：

- 最大值放在 `Property`
- 当前值放在 `Resource`

例如：

- `MaxHp` + `Hp`
- `MaxMp` + `Mp`
- `MaxStamina` + `Stamina`

## 五、初始化顺序说明

当前项目里的初始化顺序是：

1. `CombatEntity.Awake()` 中先创建 `properties`
2. 在 `CombatEntity.Awake()` 中先注册 `Hp` 资源
3. 在 `Player.Start()` 或 `Enemy.Start()` 中再注册 `MaxHp` 等属性

这意味着一个很重要的注意点：

当前实现下，注册完 `MaxHp` 后，`Hp.MaxValue` 会同步更新，但 `Hp.Value` 不一定自动变成满值。

因此在初始化完成后，建议显式调用一次：

```csharp
combatEntity.hp.Reset();
```

这样可以确保角色开局是满血。

如果以后你增加 `Mp`、`Stamina`，也建议在初始化完成后手动 `Reset()` 一次。

## 六、最小案例

下面给出一个基于当前实现的最小生命系统示例。

### 1. 注册属性

```csharp
var properties = combatEntity.properties;

properties.BeginBatch();
properties.RegisterProperty(PropertyType.MaxHp, 100, 1, 999999);
properties.RegisterProperty(PropertyType.Attack, 10, 0, 999999);
properties.RegisterProperty(PropertyType.Defense, 10, 0, 999999);
properties.EndBatch();
```

这一步的含义是：

- 注册最大生命值属性 `MaxHp`
- 注册攻击和防御属性

### 2. 初始化当前生命值

```csharp
combatEntity.hp.Reset();
```

执行后：

- `combatEntity.hp.MaxValue == 100`
- `combatEntity.hp.Value == 100`

### 3. 扣血

```csharp
combatEntity.hp.Minus(25);
```

执行后：

- `combatEntity.hp.MaxValue == 100`
- `combatEntity.hp.Value == 75`

### 4. 回血

```csharp
combatEntity.hp.Add(10);
```

执行后：

- `combatEntity.hp.Value == 85`

如果回复超过上限，会被自动截断到 `MaxValue`。

### 5. 读取当前值与最大值

```csharp
int currentHp = combatEntity.hp.Value;
int maxHp = combatEntity.hp.MaxValue;
float hpPercent = combatEntity.hp.Percent;
```

## 七、最小 Buff 示例

下面示例表示给角色加一个攻击力 Buff：

```csharp
var source = new ModifierSource(ModifierSourceType.Buff, 1001);

ModifierHandle handle = combatEntity.properties.AddModifier(
    PropertyType.Attack,
    20,
    ModifierType.Add,
    source);

int finalAttack = combatEntity.properties.GetFinalValue(PropertyType.Attack);
```

如果原始攻击力是 `10`，那么此时：

- `finalAttack == 30`

移除 Buff：

```csharp
combatEntity.properties.RemoveModifier(PropertyType.Attack, handle);
```

或者按来源统一移除：

```csharp
combatEntity.properties.RemoveModifiersBySource(source);
```

## 八、如果以后要扩展魔法值

如果以后你要做魔法值，推荐继续沿用同一套模式：

### 1. 新增属性

- `PropertyType.MaxMp`

### 2. 新增资源

- `ResourceType.Mp`

### 3. 注册并初始化

```csharp
properties.RegisterProperty(PropertyType.MaxMp, 50, 0, 999999);
var mp = properties.RegisterResource(ResourceType.Mp, PropertyType.MaxMp);
mp.Reset();
```

### 4. 读取魔法值

```csharp
int currentMp = mp.Value;
int maxMp = mp.MaxValue;
float mpPercent = mp.Percent;
```

## 九、推荐使用规范

为了避免后续系统混乱，建议统一按下面的方式使用：

- 不要把“当前生命值”做成 `Property`
- 不要把“最大生命值”做成 `Resource`
- 所有最大值都放进属性系统
- 所有当前剩余量都放进资源系统
- Buff、装备、技能修正优先改 `Property`
- 受伤、回血、耗蓝优先改 `Resource`

## 十、一句话总结

这套系统可以简单理解为：

- `Property` 管“角色理论上有多强”
- `Resource` 管“角色当前还剩多少”

例如：

- `MaxHp` 是属性
- `Hp.Value` 是当前生命值

这就是当前战斗数值系统最核心的结构。

## 十一、派生属性与链式属性

### 1. 什么是派生属性

派生属性是指它的基础值不是手动填写的，而是由其他属性的最终值计算得到。

例如：

- `DamageReductionRate = AllRound * 0.5`
- `MaxHp = Constitution * 10 + EquipmentHpBonus`

在当前框架中，这类属性通过 `RegisterDerivedProperty(...)` 注册。

### 2. 派生属性的结算顺序

派生属性固定按下面的顺序结算：

1. 先读取依赖属性的最终值
2. 用公式计算出派生属性的基础值
3. 再叠加该派生属性自己的 `Modifier`
4. 得到最终值 `FinalValue`

这意味着：

- 上游属性吃到 Buff 后，下游派生属性会自动重算
- 派生属性本身也可以继续挂 Buff、装备、技能修正

### 3. 派生属性和普通属性的区别

- 普通属性可以用 `SetBaseValue(...)` 直接设置基础值
- 派生属性不允许 `SetBaseValue(...)`
- 派生属性调用 `GetBaseValue(...)` 时，返回的是公式算出来的基础值

### 4. 派生属性的注册要求

注册派生属性时，必须满足：

- 依赖属性必须已经注册
- 依赖列表不能为空
- 不能依赖自己
- 不允许形成循环依赖

也就是说，注册顺序必须遵守“先上游，后下游”。

### 5. 最小派生属性示例

下面是一个最小链式属性示例：

```csharp
var properties = combatEntity.properties;

properties.RegisterProperty(PropertyType.AllRound, 100, 0, 999999);
properties.RegisterDerivedProperty(
    PropertyType.DamageReductionRate,
    new[] { PropertyType.AllRound },
    context => context.GetFinalValue(PropertyType.AllRound) / 2,
    0,
    10000);
```

这段代码表示：

- `AllRound` 是普通属性
- `DamageReductionRate` 是派生属性
- 它的基础值等于 `AllRound` 最终值的一半

例如：

```csharp
int damageReduction = properties.GetFinalValue(PropertyType.DamageReductionRate);
```

如果此时：

- `AllRound = 100`

那么：

- `DamageReductionRate = 50`

### 6. 上游属性变化时的效果

如果你给 `AllRound` 加一个 Buff：

```csharp
properties.AddModifier(
    PropertyType.AllRound,
    20,
    ModifierType.Add,
    new ModifierSource(ModifierSourceType.Buff, 2001));
```

那么：

- `AllRound` 会先变成 `120`
- `DamageReductionRate` 会自动重算成 `60`

### 7. 派生属性自身继续吃 Modifier

如果你再给 `DamageReductionRate` 自己加一个 Buff：

```csharp
properties.AddModifier(
    PropertyType.DamageReductionRate,
    10,
    ModifierType.Add,
    new ModifierSource(ModifierSourceType.Buff, 2002));
```

那么最终结果会变成：

- 先由公式得到 `60`
- 再叠加自身 `+10`
- 最终 `DamageReductionRate = 70`

### 8. 多级链式属性

框架也支持多级链式：

```csharp
A -> B -> C
```

例如：

- `B` 依赖 `A`
- `C` 依赖 `B`

当 `A` 变化时：

- `B` 会自动重算
- `C` 也会继续自动重算

### 9. 派生属性与资源绑定

派生属性也可以拿来作为资源最大值。

例如：

- `MaxHp` 可以是普通属性
- 也可以是派生属性

只要资源绑定的是一个可计算的最终属性，资源最大值就会同步更新。
