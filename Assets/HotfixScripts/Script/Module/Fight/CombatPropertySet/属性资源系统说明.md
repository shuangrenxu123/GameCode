# 战斗属性与资源框架说明（CombatPropertySet）

## 1. 框架定位

本框架将战斗数值拆成两层：

- `Property`（属性）：用于表示“可计算、可被 Modifier 影响的最终能力值”，例如攻击、防御、最大生命等。
- `Resource`（资源）：用于表示“当前可消耗/可恢复的动态值”，例如当前生命、当前护盾。

这两层通过统一门面 `CombatPropertySet` 管理，但内部职责分离：

- 属性计算图（`StatNode`）负责依赖、脏标记、重算、Modifier。
- 资源图（`ResourceNode`）负责资源对象及其上限策略。

## 2. 核心能力

1. 支持属性与资源并存。
2. 支持派生属性（某属性由其他属性公式计算得到）。
3. 支持两类资源：
   - 属性绑定资源（如 `Hp` 绑定 `MaxHp`）。
   - 独立资源（如护盾，不依赖属性存在）。
4. 支持 Modifier 的新增、更新、激活/停用、移除、按来源批量移除。
5. 支持批处理（`BeginBatch/EndBatch`）以降低频繁变更成本。

## 3. 关键对象职责

- `CombatPropertySet`
  - 对外统一入口。
  - 维护属性节点与资源节点。
  - 负责脏标记传播、重算调度、资源同步。

- `StatNode`
  - 单个属性节点，统一承载普通属性与派生属性状态。
  - 派生属性通过 `Dependencies + Formula` 计算基础值。

- `ResourceNode`
  - 单个资源节点，持有 `ResourceValue` 与上限策略 `IResourceMaxPolicy`。

- `IResourceMaxPolicy`
  - 资源最大值来源策略接口。
  - `PropertyBoundMaxPolicy`：最大值来自某个属性最终值。
  - `FixedMaxPolicy`：最大值为独立可设置值。

- `ResourceValue`
  - 资源当前值/最大值容器。
  - 提供增减、重置、百分比和事件通知。

## 4. 关键机制

### 5.1 普通属性与派生属性

- 普通属性：可通过 `SetBaseValue` 修改基础值。
- 派生属性：基础值由 `Formula` 计算，禁止 `SetBaseValue`。
- 派生属性注册时会做：
  - 依赖已注册校验。
  - 重复依赖校验。
  - 环路校验。

### 5.2 Modifier 结算顺序

在一个属性上，激活的 Modifier 按下列逻辑合成：

1. `Add`：基础加减。
2. `AddPercent`：百分比叠加。
3. `Multiply`：乘算因子连乘。
4. `Override`：若存在则直接覆盖最终值。

最终值会被 `StatDefinition` 的 `MinFinalValue/MaxFinalValue` 裁剪。

### 5.3 脏标记与批处理

- 属性变化后，当前节点及其依赖链下游节点被标记为脏。
- 非批处理模式：立即级联重算。
- 批处理模式：延迟到 `EndBatch` 一次性重算。

### 5.4 资源最大值同步

- 绑定资源：当绑定属性最终值变化时，资源 `MaxValue` 自动同步。
- 独立资源：不受属性变化影响，只能通过 `SetResourceMax` 主动修改上限。

## 5. 对外 API 速查

### 6.1 属性 API

- `RegisterProperty(...)`
- `RegisterDerivedProperty(...)`
- `IsRegistered(...)`
- `SetBaseValue(...)`
- `GetBaseValue(...)`
- `GetFinalValue(...)`
- `TryGetFinalValue(...)`

### 6.2 Modifier API（仅作用于属性）

- `AddModifier(...)`
- `UpdateModifierValue(...)`
- `RemoveModifier(...)`
- `SetModifierActive(...)`
- `RemoveModifiersBySource(...)`

### 6.3 资源 API

- `RegisterPropertyBoundResource(...)`
- `RegisterStandaloneResource(...)`
- `TryGetResource(...)`
- `SetResourceMax(...)`（仅独立资源可成功）

### 6.4 批处理 API

- `BeginBatch()`
- `EndBatch()`

## 6. 使用示例

### 7.1 绑定资源（生命）

```csharp
properties.RegisterProperty(PropertyType.MaxHp, 100, 1, 999999);
var hp = properties.RegisterPropertyBoundResource(ResourceType.Hp, PropertyType.MaxHp);
hp.Reset();
```

### 7.2 独立资源（护盾）

```csharp
var shield = properties.RegisterStandaloneResource(ResourceType.Shield, 300, resetCurrent: true);
shield.Minus(50);          // 护盾受伤
properties.SetResourceMax(ResourceType.Shield, 500); // 调整护盾上限
```

### 7.3 派生属性

```csharp
properties.RegisterProperty(PropertyType.AllRound, 100, 0, 999999);
properties.RegisterDerivedProperty(
    PropertyType.DamageReductionRate,
    new[] { PropertyType.AllRound },
    ctx => ctx.GetFinalValue(PropertyType.AllRound) / 2,
    0,
    100);
```

### 7.4 Modifier

```csharp
var source = new ModifierSource(ModifierSourceType.Buff, 2001);
ModifierHandle handle = properties.AddModifier(
    PropertyType.Attack,
    20,
    ModifierType.Add,
    source);

properties.SetModifierActive(PropertyType.Attack, handle, false);
properties.RemoveModifier(PropertyType.Attack, handle);
```

## 7. 约束与建议

- 已移除旧接口 `RegisterResource(ResourceType, PropertyType)`，请统一使用新资源 API。
- Modifier 当前只作用于属性，不作用于资源。
- 派生公式建议只读取属性（通过 `DerivedPropertyContext`）。
- 初始化阶段建议先注册属性，再注册绑定资源，最后 `Reset()` 当前值。
